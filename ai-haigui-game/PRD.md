# AI 海龟汤游戏产品需求文档 (PRD)

## 1. 产品概述
- **项目名称**：AI 迷雾 (AI Mist)
- **产品定义**：一个由大语言模型（LLM）驱动的互动推理网站。AI 担任“汤主”，负责主持、判定和引导，玩家通过限制性提问还原故事真相。
- **核心价值**：突破传统海龟汤需要真人主持的限制，实现 24/7 随时推理。

---

## 2. 核心业务流程

1. **选择剧本**：玩家在大厅选择一个“汤面”（故事开头）。
2. **互动推理**：玩家输入提问（如：“死者是自杀吗？”）。
3. **AI 判定**：AI 根据后台隐藏的“汤底”逻辑，仅回复【是/否/无关】。
4. **揭晓真相**：玩家认为推理完成时，点击查看汤底，或由 AI 判定还原度。

---

## 3. 功能模块详细说明

### 3.1 游戏大厅 (Game Lobby)
- **卡片布局**：展示故事标题、难度（1-5星）、分类标签（如：悬疑、惊悚、温馨）。
- **进度存档**：本地存储玩家已通关的题目。

### 3.2 游戏核心页 (Game Room)
- **对话流界面**：
  - 顶部固定显示“汤面”内容。
  - 中间为气泡式对话历史。
  - 底部为提问输入框。
- **智能约束**：AI 的回答被严格锁定在“是”、“否”、“与真相无关”三个维度。
- **提示系统**：当对话超过 20 轮且无进展时，激活“灵感提示”按钮。

### 3.3 结算页面 (Result Page)
- **真相大白**：展示完整的故事背景（汤底）。
- **复盘评价**：AI 根据对话记录，评估玩家的洞察力等级（如：名侦探、普通市民、路人乙）。

---

## 4. 技术栈建议
- **前端框架**：React 18 + TypeScript
- **样式方案**：Tailwind CSS (深色主题：Slate/Zinc 系列)
- **AI 模型**：DeepSeek-V3 / Claude 3.5 Sonnet (侧重逻辑推理)
- **部署平台**：Vercel

## 5. 核心 Prompt (系统提示词) 参考

```text
# Role
你是一个冷酷、客观的海龟汤游戏主持人。

# Rules
1. 你掌握着故事的最终真相（汤底）。
2. 玩家会根据故事开头（汤面）向你提问。
3. 你的回答必须严格遵循：如果玩家猜对了部分真相，回“是”；如果错误，回“否”；如果与逻辑无关，回“无关”。
4. 严禁直接输出真相中的任何名词，直到玩家点击结束。
